Git Push

Regardless of whether you started your project on GitHub and cloned it to your local computer or if you initialized your project locally and then pushed it to your GitHub account, the process of syncing between local and remote is not automatic.

If you are working alone on your project, the most common workflow to keep things in sync is:

  • Work on your files locally.
  • Commit your changes.
  • Push those changes to the remote repo on GitHub

To sync your committed changes from local to remote, use the following command:

  • git push 

You will be prompted for your username and passphrase.
