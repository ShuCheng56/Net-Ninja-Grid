Excluding Files and Folders

There are various situation where you actually don't want git to track versions of certain files or folders within your project, such as:

  • system files created by your OS
  • node_modules folder
    • you'll learn about this when we get to React
  • computer generated files that are created/updated through a compile program 
    • you'll learn about this when we get to Sass and Gulp

You can instruct git to ignore such files and folders by creating a simple text file, appropriately name .gitignore, in the root of your project.

Note to Mac users: 

  • By default MacOS hides files that begin with "."
  • To toggle the visibility of hidden files in Finder, press cmd+shift+.

Once you have created your .gitignore file, open it with a text editor and list them out, one per line, as follows:

  • Ignore a single file
    • file-to-ignore.txt (or whatever file extension is appropriate)
  • Ignore a directory
    • folder-name/

You can also use * as a wildcard to exclude groups of files, for instance the following would exclude all pdf files in a directory named resources:

  • resources/*.pdf

To add comments in your .gitignore file, start the line with #
