Merging Branches

When you're satisfied that your new or edited feature branch is ready for production, you will want to (carefully) merge it back into the main branch. 

If you are working solo on your project this process will look like:

  • commit all changes on your feature branch
  • switch to the main branch with:
    • git checkout main
  • merge the commits from your feature branch into main with:
    • git merge [feature branch name] 
  • push the complete repository to GitHub
    • git push

If, however, you are working on a team and a review process would be more appropriate before integrating your new feature:

  • commit all changes on your feature branch
  • push your feature branch to GitHub
    • git push origin [feature branch name]
  • Then, on GitHub, issue a Pull Request indicating which branch is to be reviewed before being merged into main
  • Once your team is satisfied that your feature is approved, click Merge Pull Request, leave a comment as to what is being merged and finally, click Confirm Merge.
  • You may then choose to delete the feature branch (to keep things clean).
