Committing Changes

Once you've added your files (and instructed git which files and folders to ignore), you must commit them, including a message as to what has changed, to create a snapshot of that revision of your project.

Once you've performed a commit it is added to the version history of project and you can, if needed, revert your project to that point in time as well as compare what has changed between any two revisions.

To commit your changes, run the following command:

  • git commit –m "Describe your changes"

Note: Good commit messages are essential for maintaining version history, particularly if you need to revert to a previous commit.
